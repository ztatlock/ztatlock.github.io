# Publication Artifact Follow-Up

This note captures the current state of the publication-artifact cleanup so it
is easy to resume later without re-deriving context.

## Agreed Policy

- Public talk links on the website should normally point to the canonical
  public watch URL, usually YouTube.
- Local archival copies of talks should live in `~/Desktop/WEBFILES`.
- Small first-party artifacts that the site serves directly should live in the
  repo under `site/pubs/<slug>/`.
- Large source artifacts and preservation copies should live in `WEBFILES`.
- Missing required repo publication artifacts are errors.
- Missing slide PDFs and poster PDFs are warnings until manually curated.
- A publication with no public talk link is informational by itself, not an
  error.

The current canonical policy doc is:

- `docs/policy/publication-artifacts.md`
